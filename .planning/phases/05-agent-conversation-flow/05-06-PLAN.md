## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06-01 | I (XSS via agent-generated content) | TimelineTab renders response_text | mitigate | React auto-escapes text children; render agent text as `{item.data.metadata.response_text}` — never `dangerouslySetInnerHTML` |
| T-05-06-02 | T (CSRF on move endpoint via SuggestedStageButton) | SuggestedStageButton onClick | mitigate | use existing card-move POST which already requires JWT cookie + CSRF token per app convention |
| T-05-06-03 | I (Timeline fetched for other tenant via URL tampering) | TimelineTab fetch | mitigate | Backend enforces tenant check (05-05 T-05-05-01); frontend relies on server rejection — no mitigation required client-side |
| T-05-06-04 | E (SDR moves card to stage it shouldn't) | SuggestedStageButton → move endpoint | accept | existing move endpoint already enforces pipeline/stage ownership; button is convenience only |
| T-05-06-05 | R (Stale read in QualifiedBadge after SDR confirms) | KanbanBoard | accept | badge refreshes on next board poll / realtime notification (existing NotificationStream); flicker window is acceptable |
| T-05-06-06 | I (Advanced numeric inputs bypass client clamp) | agent editor form | mitigate | backend (05-01 AgentPromptProfile DTO) clamps historyWindow to [10,50] and summaryThreshold to [5,20] — client is convenience |
</threat_model>
