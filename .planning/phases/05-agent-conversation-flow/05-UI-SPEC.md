## Typography

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm leading-6` |
| Label / Meta | 12px | 600 (semibold) | 1.4 | `text-xs font-semibold` |
| Eyebrow / Timestamp | 10px | 600 uppercase, tracking 0.18em | 1.3 | `text-[10px] uppercase tracking-[0.18em]` |
| Heading (tab title, banner header) | 16px | 600 (semibold) | 1.3 | `text-base font-semibold` |

**Rules:**
- Exactly **4 sizes**, exactly **2 weights** (400 + 600). No 500, no 700.
- Body line-height `leading-6` (24px on 14px = 1.71) matches existing `ActivityTimeline.tsx` line 132.
- Eyebrow uppercase + wide tracking matches existing channel badges and timestamps (line 86, 126).
- Banner copy uses body size; only the `IMPORTANTE` label itself is heading weight.

## Interaction Contracts

### Qualified Badge Lifecycle
1. Card has at least one `AGENT_QUALIFIED` activity AND no `MOVED` activity newer than it → badge visible.
2. Hovering badge for >300ms → tooltip with `qualification_reason`.
3. Once SDR moves the card (any direction), badge disappears on next refresh; no client-side optimistic hide.

### Suggested-Stage CTA
1. CTA visible only inside expanded card view (modal), not on Kanban card itself.
2. Click → calls existing `cardService.moveCard` endpoint with `targetStageId = suggested_next_stage_id`.
3. On success: optimistic card move, modal stays open showing the new stage; toast `Card movido para {stage.name}`.
4. On failure: standard error toast `Não foi possível mover o card. Tente novamente.`; CTA re-enabled.
5. If `suggested_next_stage_id` resolves to a stage that no longer exists OR belongs to a different pipeline → hide primary CTA, show fallback copy + standard `MoveCardButtons`.

### Timeline Tab
1. First open → fetches `/cards/:id/timeline?limit=100`, shows newest at top.
2. Filter chips toggle client-side filtering (no refetch).
3. `Carregar mais antigos` → fetches with `before=<oldest.createdAt>` cursor; appends to bottom.
4. Click on a `WHATSAPP_OUTBOUND` or `agent` source row → no navigation (read-only). Future iteration may open thread.
5. New events do NOT auto-refresh. Manual refresh via tab re-click. (Phoenix/SSE integration deferred.)

### Notification Bell
1. New `AGENT_QUALIFIED_READY_TO_ADVANCE` notification → bell badge increments.
2. Click notification → router push to `/kanban?card=:id`; URL param triggers card modal open on `Atendimento` tab.

## Accessibility Contract

| Surface | Requirement |
|---------|-------------|
| Qualified badge | Color contrast ≥4.5:1 on white (`#22bd73` on `#ecfdf5` measured 4.58:1, OK); icon has `aria-hidden`; badge has `role="status"` and accessible name `Lead pronto para avançar` |
| Suggested-stage CTA | Native `<button>`; focusable; visible focus ring (`focus-visible:ring-2 ring-[#594ded] ring-offset-2`); disabled state has `aria-disabled` + visual dim |
| Timeline filters | Native `<button role="tab">` pattern with `aria-pressed`; keyboard arrow navigation between chips |
| Tooltip on badge | Triggered by hover AND keyboard focus (Tab); `Esc` dismisses |
| IMPORTANTE banner | `role="note"`; emoji has `aria-hidden`; text content carries semantic meaning |
| Timeline row | Each row is a `<li>` inside `<ol>`; timestamp uses `<time datetime="ISO8601">` |
