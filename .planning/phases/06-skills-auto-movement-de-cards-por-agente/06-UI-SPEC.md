## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tag padding (`px-1`, `py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`, `px-4`) |
| lg | 24px | Section padding (`gap-6`, `p-6`) |
| xl | 32px | Layout gaps (`p-8` on lg breakpoint) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level hero spacing (`py-20`) |

Exceptions:
- Skill drag handle touch target: 40px minimum (use `h-10 w-10` container, matching existing timeline dot pattern)
- Toggle switch: 44px tall tap target for movementMode toggle (wrap in `py-3` row)

Source: Extracted from existing component patterns in `agentes/page.tsx` and `ActivityTimeline.tsx`

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-6 or leading-7) |
| Label / eyebrow | 11px (text-[11px]) | 400 (normal) | 1.0 — uppercase tracking-[0.28em] |
| Subheading | 18px (text-lg) | 600 (semibold) | 1.2 |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 |

Note: Label/eyebrow visual differentiation is achieved through `uppercase` + `tracking-[0.28em]` treatment — no additional weight needed. Skill content preview in AgentSkillItem rows uses 14px body size (`text-sm`) with `text-slate-500` and `line-clamp-1` to visually differentiate from primary body text without introducing an additional type size. Maximum 4 sizes enforced.

Source: Directly observed from `agentes/page.tsx` — reuse existing pattern. Do NOT introduce new type sizes.

## Color

|------|-------|-------|

| Success | `#22bd73` (`--accent-success`) | auto_moved_by_agent timeline icon, auto-move enabled state indicator |

| Destructive | `#fc736f` (`--accent-danger`) | Delete skill button (rose-700 text, rose-200 border, rose-50 bg) |

Accent (`#594ded`) reserved for:
- Icon on compiled prompt preview panel (`text-[#594ded]`)
- Focus ring on inputs/textareas (`focus:border-[#594ded]/35`)
- Drag handle grip icon color in skill order list
- `RULE_STEP_SENT` timeline entry (existing `text-[#594ded]`)

CTA cyan (`bg-cyan-300`) reserved for:
- "Nova skill" primary button on `/agentes/skills` page
- "Salvar skill" button in skill modal
- Active wizard step indicator pills (existing pattern)

Source: `globals.css` CSS variables + observed class usage across `agentes/page.tsx`, `ActivityTimeline.tsx`

## Interaction States

### Skill CRUD (library page)

| State | Visual |
|-------|--------|
| Loading skills | `<p className="text-sm text-slate-400">Carregando skills...</p>` in list area |
| Empty library | Dashed card with Sparkles icon, heading "Nenhuma skill criada ainda.", body "Crie a primeira skill para injetar contexto especializado nos agentes do workspace." + "Nova skill" CTA button |
| Saving skill | Button shows "Salvando..." text + disabled + `opacity-60` |
| Save success | Inline feedback: `border-emerald-200 bg-emerald-50 text-emerald-700 rounded-2xl px-4 py-3 text-sm` (same as `operationFeedback` pattern in agentes page) |
| Delete skill | Confirmation: inline `window.confirm` dialog — "Excluir esta skill remove o vínculo de todos os agentes que a usam. Continuar?" (no modal, direct confirm dialog) |
| Delete error | `border-rose-200 bg-rose-50 text-rose-700` feedback banner |

### Agent wizard — Skills step

| State | Visual |
|-------|--------|
| No skills in tenant | Disabled "Adicionar skill" button with hint: "Crie skills na biblioteca antes de adicioná-las ao agente." |
| Skills list with items | Ordered draggable list; drag in progress shows `opacity-60` on dragged item |
| Remove skill from agent | X button on each row with `aria-label="Remover skill"`; no confirmation (low-risk operation — order is saved on wizard submit) |

### movementMode toggle

| State | Visual |
|-------|--------|
| MANUAL (default) | Checkbox unchecked; label "Mover card automaticamente (AUTO)"; helper: "Modo padrão: o SDR confirma o avanço." |
| AUTO active | Checkbox checked; warning banner appears below: amber left-border aside explaining 3 gates |
| No agent assigned | Toggle row hidden entirely; no empty state shown (tab/section not rendered) |
| Saving mode change | Save button in drawer shows "Salvando..." |

## Stage Drawer — Auto-move Tab/Section

The movementMode toggle is added as a new tab "Auto-move" in the existing `StageRuleDrawer` (`TabId` extended to include `'auto-move'`). The tab renders only when `assignedAgent !== null`; when no agent is assigned, the tab is hidden entirely.

```
Tab: "Auto-move" (hidden if no agent assigned to stage)
├── [amber aside banner — always shown when tab visible]
│   "Quando ativo, o agente move o card automaticamente..."
├── Toggle row
│   ├── <label> — inline-flex items-center gap-3 rounded-2xl border bg-slate-50 px-4 py-3
│   ├── <input type="checkbox"> — checked = AUTO, unchecked = MANUAL
│   └── Label text + helper text
└── Save button — "Salvar configuração" (bg-cyan-300, full-width on mobile)
```
