## Interaction Contracts

### RĂ©gua pause/resume (CardDetailSheet)

- Button click â†’ optimistic UI toggle (swap icon + label instantly)
- API call in background; on failure â†’ revert state + show inline error text below button
- Busy state: show `<Loader2 className="h-3.5 w-3.5 animate-spin" />` inside button (matches existing `busyCardId` pattern)
- Transition: `transition` class only (no custom duration â€” matches existing card pattern)

### Agente takeover toggle (CardDetailSheet)

- Single click on "Assumir conversa" â†’ POST to toggle endpoint
- Button disabled when `!activeConversation` (no conversation exists)
- Status badge updates immediately after success
- Uses existing `getConversationMeta()` + `conversationTone` mapping

### Stage rule/agent config (StageRuleDrawer modal tabs)

- Inline edit: clicking any step row puts it in edit mode (replaces row with form, same pattern as `StageTemplatesModal`)
- Save: `PATCH` call with optimistic list update
- Cancel: reverts to display row
- Tab switching: no API call â€” just renders different tab content
- Unsaved changes on tab switch: no blocking guard (keep it simple; tabs only switch config areas)

### Meta form mapping (configuracoes)

- "Adicionar mapeamento" opens inline form row (not a modal â€” same dashed-add pattern)
- Pipeline selector â†’ Stage selector cascade (stage dropdown populates after pipeline selected)
- Meta Form ID: free-text input with placeholder "ex: 123456789"
- Save: POST call; success shows row in list; error shows inline error below form

### Drag-and-drop card movement (existing â€” rĂ©gua cancellation side-effect)

- No UI change needed: cancellation of prior rĂ©gua jobs is backend-only
- Card dragged to new stage shows existing stage-theme color during drag (already implemented)
- After drop: card badge updates from "RĂ©gua ativa" to "RĂ©gua na fila" (handled by `getRuleSummary()` via refresh)
