- `async handleVerification(mode: string, token: string, challenge: string)` — if mode==='subscribe' and validateToken(token) → return { ok: true, challenge }; else { ok: false }. Controller handles the raw response.
- `async ingestLead(payload: MetaLeadPayload): Promise<void>` — iterate entries, for each change with field==='leadgen' call `processLead(change.value.form_id, change.value.leadgen_id)`. Wrap each processLead in try/catch so one failure doesn't block others.
- `private async processLead(formId: string, leadgenId: string): Promise<void>`:
  1. `const mapping = await prisma.metaWebhookMapping.findFirst({ where: { metaFormId: formId }, include: { stage: true } })`. If null, return.
  2. `try { await prisma.metaLeadIngestion.create({ data: { tenantId: mapping.tenantId, metaLeadId: leadgenId } }) } catch (e) { if (isPrismaUniqueViolation(e)) return; throw; }` — idempotency gate.
  3. Fetch lead details: if `mapping.pageAccessToken`, try `fetch('https://graph.facebook.com/v19.0/' + leadgenId + '?fields=field_data&access_token=' + mapping.pageAccessToken)`. Parse field_data for name/phone/email. On any failure, fall through with `name='Lead Meta'`, phone/email undefined, notes='Campos não recuperados'.
  4. `const contact = await this.upsertContact(mapping.tenantId, phone, email, name)`
  5. `const card = await this.cardService.create(mapping.tenantId, { title: name || 'Lead Meta', stageId: mapping.stageId, contactId: contact.id, customFields: { source: 'meta-ads', metaLeadId: leadgenId, notes: extraFields } })`
  6. `await prisma.metaLeadIngestion.update({ where: { metaLeadId: leadgenId }, data: { cardId: card.id } })`
  7. `await prisma.cardActivity.create({ data: { cardId: card.id, type: 'META_LEAD_INGESTED', content: 'Lead recebido via Meta Lead Ads (form ' + formId + ')' } })`

**Create `apps/api/src/meta-webhook/meta-webhook.controller.ts`:**

```typescript
import { Body, Controller, Delete, Get, Param, Post, Query, Req, Res, UseGuards } from '@nestjs/common';
import type { Response } from 'express';
import { MetaWebhookService } from './meta-webhook.service';
import { CreateMetaMappingDto } from './dto/create-meta-mapping.dto';
import { JwtAuthGuard } from '../auth/jwt-auth.guard';
import { TenantGuard } from '../auth/tenant.guard';
import { TenantId } from '../auth/tenant-id.decorator';

@Controller()
export class MetaWebhookController {
  constructor(private readonly service: MetaWebhookService) {}

  // Public — Meta sends GET /meta-webhook for verification
  @Get('meta-webhook')
  public async verifyWebhook(
    @Query('hub.mode') mode: string,
    @Query('hub.verify_token') token: string,
    @Query('hub.challenge') challenge: string,
    @Res() res: Response,
  ): Promise<void> {
    const result = await this.service.handleVerification(mode, token, challenge);
    if (result.ok) {
      res.status(200).type('text/plain').send(challenge);
    } else {
      res.status(403).type('text/plain').send('Forbidden');
    }
  }

  // Public — Meta sends POST /meta-webhook for new leads
  @Post('meta-webhook')
  public async receiveLead(@Body() payload: any, @Res() res: Response): Promise<void> {
    // Return 200 immediately to avoid Meta retry storms; process async
    res.status(200).type('text/plain').send('EVENT_RECEIVED');
    this.service.ingestLead(payload).catch((err) => {
      // Log but do not re-throw — Meta should not retry for app-side errors
      console.error('[MetaWebhook] ingestLead failed:', err);
    });
  }

  // Authenticated — mapping CRUD
  @Get('meta-mappings')
  @UseGuards(JwtAuthGuard, TenantGuard)
  public async list(@TenantId() tenantId: string) {
    return this.service.listMappings(tenantId);
  }

  @Post('meta-mappings')
  @UseGuards(JwtAuthGuard, TenantGuard)
  public async create(@TenantId() tenantId: string, @Body() dto: CreateMetaMappingDto) {
    return this.service.createMapping(tenantId, dto);
  }

  @Delete('meta-mappings/:id')
  @UseGuards(JwtAuthGuard, TenantGuard)
  public async remove(@TenantId() tenantId: string, @Param('id') id: string) {
    return this.service.deleteMapping(tenantId, id);
  }
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-06 | Spoofing | POST /meta-webhook (forged lead) | mitigate | processLead returns silently if no MetaWebhookMapping matches form_id (no card created); responses always 200 to prevent attacker probing |
| T-2-07 | Tampering | Webhook replay / duplicate delivery | mitigate | MetaLeadIngestion.metaLeadId @unique — Prisma unique violation triggers idempotent skip |
| T-2-08 | Information Disclosure | listMappings response | mitigate | `select` clause explicitly omits pageAccessToken; verified via acceptance criterion grep |
| T-2-09 | Denial of Service | Webhook POST flood | mitigate | Return 200 immediately (fire-and-forget ingestLead); processing failures do not trigger Meta retries |
| T-2-10 | Information Disclosure | Meta pageAccessToken stored in DB | mitigate | Stored server-side only; never included in API responses; access requires admin tenant context |
