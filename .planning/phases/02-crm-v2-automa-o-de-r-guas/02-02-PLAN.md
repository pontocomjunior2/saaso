**Create `apps/api/src/stage-rule/stage-rule.service.ts`** — inject `PrismaService`, `StageRuleQueueService` (forwardRef), `TenantService` (for businessHours). Methods:

- `getRuleForStage(tenantId: string, stageId: string)` — `prisma.stageRule.findFirst({ where: { tenantId, stageId }, include: { steps: { orderBy: { order: 'asc' } } } })`
- `createRuleForStage(tenantId: string, stageId: string, dto)` — validate stage belongs to tenant; create rule (throws ConflictException if exists via catch on unique violation)
- `updateRule(tenantId, ruleId, dto)` — findFirst+throw NotFoundException if not found; update isActive
- `replaceSteps(tenantId, ruleId, dto)` — in tx: deleteMany existing steps, createMany new steps ordered
- `deleteRule(tenantId, ruleId)` — cascade delete via Prisma onDelete
- `startRuleRun(cardId, stageId, tenantId, triggerSource)`:
  1. Load rule + steps for stage
  2. If no rule OR !isActive, return null
  3. Load tenant.featureFlags.businessHours
  4. Create StageRuleRun (status=RUNNING, startedAt=now)
  5. For each step, compute scheduledFor = `nextBusinessHour(new Date(now + step.dayOffset*86400000), businessHours)`, create StageRuleRunStep (status=PENDING)
  6. Call `stageRuleQueueService.enqueueRuleStep(runStep.id, runStep.scheduledFor)` for each step
  7. Return the run
- `pauseRun(runId, tenantId)` — update status=PAUSED, pausedAt=now. For each PENDING step, attempt `queueService.removeJob(step.id)` (best effort, swallow errors).
- `resumeRun(runId, tenantId)` — load run, find PENDING steps, for each: recompute scheduledFor = `nextBusinessHour(new Date(now + step.dayOffset*86400000), businessHours)` (using original step.dayOffset from StageRuleStep reference), update step.scheduledFor, re-enqueue. Set run.status=RUNNING.
- `cancelRun(runId, tenantId)` — inside tx: set status=CANCELED, canceledAt=now. After tx, best-effort remove pending jobs.
- `cancelActiveRunsForCard(cardId, tenantId)` — findMany active runs (status in RUNNING|PAUSED), cancel each.
- `executeStep(stepId: string)` — called by queue worker. Load step with run + rule + template + card + contact. If run.status=CANCELED or PAUSED, set step.status=SKIPPED and return. Otherwise: set step.status=RUNNING, send via existing `WhatsappService.sendMessage` or `EmailService.sendMessage` (based on step.channel), create CardActivity with type 'RULE_STEP_SENT' content `Régua D${step.ruleStep.dayOffset} disparada via ${channel}`, set step.status=SENT + completedAt. If all steps SENT, set run.status=COMPLETED.
